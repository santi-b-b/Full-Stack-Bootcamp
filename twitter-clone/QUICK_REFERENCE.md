Clone and setup\ngit clone <repo-url>\ncd twitter-clone\n\n# 2. Install dependencies\nnpm install\n\n# 3. Setup environment\ncp .env.example .env.local\n# Edit .env.local with your MongoDB URI and Cloudinary credentials\n\n# 4. Start development server\nnpm run dev\n\n# 5. Open browser\n# Go to http://localhost:3000\n`\n\n---\n\n## 🛠️ Common Commands\n\n`bash\n# Development\nnpm run dev              # Start dev server (http://localhost:3000)\nnpm run build            # Build for production\nnpm start                # Run production build\n\n# Code Quality\nnpm run lint             # Check code with ESLint\nnpm run format           # Format code with Prettier\n\n# One-liner to dev\nnpm install && npm run dev\n`\n\n---\n\n## 🗄️ Database Setup\n\n### MongoDB Atlas (Cloud)\n1. Go to https://www.mongodb.com/cloud/atlas\n2. Create free account and cluster\n3. Get connection string\n4. Add to `.env.local` as `MONGODB_URI`\n\n### Local MongoDB\n`bash\n# Install MongoDB Community Edition\n# Windows: Use installer from mongodb.com\n# Mac: brew install mongodb-community\n# Linux: Follow docs.mongodb.com\n\n# Start MongoDB\nmongod\n\n# Connection string\nMONGODB_URI=mongodb://localhost:27017/twitter-clone\n`\n\n---\n\n## 🖼️ Image Upload Setup (Cloudinary)\n\n1. Sign up at https://cloudinary.com (free tier: 25GB)\n2. Go to Dashboard\n3. Copy your Cloud Name\n4. Go to Settings → Upload\n5. Create new upload preset (unsigned)\n6. Add to `.env.local`:\n `\n   NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your_cloud_name\n   NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET=preset_name\n   `\n\n---\n\n## 📝 File Structure at a Glance\n\n`\ntwitter-clone/\n├── src/\n│   ├── app/\n│   │   ├── api/              ← Backend routes\n│   │   ├── login/            ← Auth pages  \n│   │   └── (main)/           ← Main app pages\n│   ├── components/           ← React components\n│   ├── contexts/             ← State management\n│   └── utils/                ← Helper functions\n├── lib/                       ← Database & auth\n├── public/                    ← Static files\n├── .env.example              ← Configuration template\n├── README.md                 ← Main docs\n├── CONTRIBUTING.md           ← Contribution guide\n├── API.md                    ← API reference\n└── IMPROVEMENTS.md           ← Changes made\n`\n\n---\n\n## 🔐 Security Reminders\n\n### Do's ✅\n- ✅ Never commit `.env.local`\n- ✅ Use HTTPS in production\n- ✅ Validate all user inputs\n- ✅ Hash passwords (already done with bcrypt)\n- ✅ Use HTTP-only cookies (already configured)\n- ✅ Regenerate SESSION*SECRET for production\n\n### Don'ts ❌\n- ❌ Never put secrets in code\n- ❌ Never use NEXT_PUBLIC* for secrets\n- ❌ Never commit .env files\n- ❌ Never trust client input\n- ❌ Never log sensitive data\n- ❌ Never use weak passwords\n\n---\n\n## 🐛 Debugging Tips\n\n### Check Logs\n`bash\n# Terminal where npm run dev is running\n# Shows API errors and React warnings\n\n# Browser Console (F12)\n# Shows client-side errors and logs\n`\n\n### Common Issues\n\n**Issue:** `MONGODB_URI not defined`\n`bash\n# Solution: Check .env.local has MONGODB_URI\necho $MONGODB_URI\n# Or restart dev server\n`\n\n**Issue:** Session not persisting\n`bash\n# Solution: \n# 1.
